# Plan: gateway-zone-refactor

## 개요

스마트팜 플랫폼의 게이트웨이-구역-장치 구조를 전면 리팩토링한다.  
현재는 장치를 직접 추가하고 구역에 장치를 할당하는 구조이지만, 이를 **게이트웨이(라즈베리파이) 중심의 환경 설정** → **구역 표시** 구조로 전환한다.

---

## 현재 구조 (AS-IS)

```
Admin → 라즈베리파이 할당 → 지그비 장치 추가 (타입 지정) → 구역에 장치 직접 할당
```
- 장치를 직접 추가하고 장치 관리 페이지에서 수정/삭제
- 구역에 장치를 수동으로 할당
- 채널 맵핑은 자동화 룰에 분산
- 게이트웨이 환경 설정이 별도 페이지(GatewayEnvSettings)에 격리

## 목표 구조 (TO-BE)

```
Admin → 라즈베리파이 할당 → 게이트웨이 환경 설정 (온보드+지그비 장치 관리) → 구역에 게이트웨이 할당 → 구역에서 장치/측정기 확인 및 제어
```
- 모든 장치 관리(추가/수정/삭제/활성화)는 **게이트웨이 환경 설정**에서만 수행
- 구역은 할당된 게이트웨이의 활성화된 장치를 자동 표시
- 장치 관리 페이지 제거
- 채널 맵핑을 게이트웨이 환경 설정으로 통합

---

## 기능 요구사항

### FR-01: 게이트웨이-구역 할당

- 게이트웨이(라즈베리파이)는 **하나의 구역에만** 할당 가능
- 구역은 **여러 개의 게이트웨이**를 가질 수 있음
- 게이트웨이 페이지에서 구역 할당 가능
- 구역 페이지에서도 게이트웨이 추가 가능
- 이미 다른 구역에 할당된 게이트웨이는 재할당 시 경고 표시

### FR-02: 게이트웨이 페이지

- 게이트웨이 목록: 상태(온라인/오프라인), 할당된 구역명 표시
- 각 게이트웨이에서:
  - **원격 접속** (Web Terminal, 기존 기능 유지)
  - **환경 설정** 버튼 → GatewayEnvSettings 페이지로 이동
  - **구역 할당** 버튼/드롭다운

### FR-03: 게이트웨이 환경 설정 (GatewayEnvSettings 리팩토링)

#### 3-1. 온보드 GPIO 장치 (고정, 15개)

기본 제공 장치로 이름 변경·활성화/비활성화 가능. 삭제/순서 변경 불가.

| 번호 | 기본 이름 | 타입 |
|------|-----------|------|
| 1 | 유동팬 1번 | fan |
| 2 | 유동팬 2번 | fan |
| 3 | 유동팬 3번 | fan |
| 4 | 유동팬 4번 | fan |
| 5 | 원격제어 ON/OFF | irrigation (remote_control) |
| 6 | 액비/교반기 B접점 | irrigation (fertilizer_b_contact) |
| 7 | 1구역 관주 | irrigation (zone_1) |
| 8 | 2구역 관주 | irrigation (zone_2) |
| 9 | 3구역 관주 | irrigation (zone_3) |
| 10 | 4구역 관주 | irrigation (zone_4) |
| 11 | 5구역 관주 | irrigation (zone_5) |
| 12 | 6구역 관주 | irrigation (zone_6) |
| 13 | 7구역 관주 | irrigation (zone_7) |
| 14 | 교반기 | irrigation (mixer) |
| 15 | 액비 | irrigation (fertilizer_motor) |

- 온보드 관주 장치의 채널 맵핑은 **고정값** (변경 불가)

#### 3-2. 지그비 장치 추가 (기존 로직 유지 + 환경 설정으로 이동)

- 환경 설정 하단 "장치 추가" 버튼
- 지그비 게이트웨이에서 스캔된 장치 목록에서 선택
- 장치 타입 선택:
  - **센서(측정기)**: 온습도, CO2, 토양수분 등
  - **액추에이터**:
    - 유동팬 (fan)
    - 관수 장치 (irrigation): 8채널 / 12채널 선택, 채널 맵핑 설정
    - 개폐기 (opener): 열림/닫힘 장치 **반드시 2개 동시 선택** (인터록 필수)
- 추가 후 환경 설정에서:
  - 이름 변경
  - 삭제
  - 관수 장치: 채널별 기능 맵핑, 타이머 설정 (자동화 룰에서 이동)
  - 개폐기: 인터록 설정 확인

#### 3-3. 센서 환경 설정 (기존 유지)

- 실내외 온습도/CO2/토양 등 각 측정 역할에 어떤 지그비 장치를 맵핑할지 설정
- 기존 `env-config` 로직 그대로 유지, UI 위치만 환경 설정 탭 내로 통합

### FR-04: 구역(Zone) 페이지 리팩토링

- 구역에 할당된 게이트웨이 목록은 **표시하지 않음**
- 구역에 표시되는 항목:
  - **측정기**: 환경 설정에서 활성화된 센서 장치
  - **장치**: 환경 설정에서 활성화된 액추에이터 (온보드 + 지그비)
  - **자동제어 룰**: 구역과 연결된 룰 목록 (생성/삭제 불가, **값 편집만** 가능)
- 장치 개별 ON/OFF 제어 가능
- 예시: 온보드 15개 모두 활성화 + 지그비 온습도 1개 + 지그비 개폐기 1쌍 → 측정기 1개, 장치 17개

### FR-05: 장치 관리 페이지 제거

- `/devices` 페이지 제거 (또는 읽기 전용 뷰어로 전환, 추후 결정)
- 메뉴에서 "장치 관리" 항목 제거
- 장치 수정/삭제 기능은 게이트웨이 환경 설정에서만 제공
- 장치 목록이 필요한 경우 구역 페이지에서 확인

### FR-06: 지그비 관수 장치 채널 맵핑 통합

- 기존: 자동화 룰 페이지에서 채널 맵핑
- 변경: 게이트웨이 환경 설정의 관수 장치 카드에서 채널 맵핑
- 8채널 / 12채널 컨트롤러 지원
- 각 채널에 기능(구역 관주, 교반기 등) 할당
- 타이머 설정도 동일 위치로 이동

## 영향 범위 분석

### Backend

| 모듈 | 변경 내용 |
|------|-----------|
| `gateway-manager` | 구역 할당 API 추가 (one-to-one 제약), 게이트웨이별 활성 장치 조회 |
| `gateway-env` | 온보드 장치 기본값 시드, 장치 활성화/비활성화, 이름 변경 |
| `devices` | 장치 직접 CRUD API 제거 또는 deprecated |
| `groups` | 구역 조회 시 할당된 게이트웨이의 활성 장치 자동 포함 |
| `automation` | 채널 맵핑 API를 gateway-env로 이동 |

### Frontend

| 페이지/컴포넌트 | 변경 내용 |
|----------------|-----------|
| `GatewayManagement.vue` | 구역 할당 UI 추가, 환경 설정 버튼 유지 |
| `GatewayEnvSettings.vue` | 온보드 장치 섹션 추가, 지그비 장치 추가 플로우, 채널 맵핑 통합 |
| `Groups.vue` | 게이트웨이 목록 제거, 활성 장치/측정기/룰 표시 |
| `GroupCreation.vue` | 게이트웨이 선택 UI 유지 |
| `Devices.vue` | 페이지 제거 또는 읽기 전용 |
| `App.vue` / `router` | 장치 관리 메뉴 제거 |

### DB Schema

| 테이블 | 변경 내용 |
|--------|-----------|
| `gateways` | `house_id` (구역 할당) 유니크 제약 추가 고려 |
| `gateway_env_devices` | 온보드 장치 기본값 컬럼 추가 (`device_source`: 'onboard' \| 'zigbee', `slot_index`) |
| `devices` | 직접 관리 불필요, `gateway_env`로 역할 이전 |

## 우선순위 및 단계

### Phase 1: 게이트웨이-구역 할당 (1:N 구조 확립)
- 게이트웨이 → 구역 할당 API (one-zone 제약)
- 구역에서 게이트웨이 추가/제거 UI
- 게이트웨이 페이지에 구역 할당 UI

### Phase 2: 게이트웨이 환경 설정 리팩토링
- 온보드 GPIO 장치 15개 기본 시드 + 활성화/이름 변경
- 지그비 장치 추가/삭제/이름 변경을 환경 설정으로 통합
- 관수 채널 맵핑 UI 환경 설정으로 이동

### Phase 3: 구역 페이지 리팩토링
- 구역에서 게이트웨이 목록 제거
- 활성 장치/측정기/자동제어 룰 통합 표시
- 룰 값 편집 UI (생성/삭제 제외)

### Phase 4: 장치 관리 페이지 제거
- 메뉴 및 라우터에서 제거
- 관련 링크 정리
