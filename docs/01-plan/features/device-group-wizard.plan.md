## 개요

| 항목 | 내용 |
|------|------|
| Feature | device-group-wizard |
| Phase | Plan |
| 작성일 | 2026-03-01 |
| 우선순위 | Medium |

## 배경 및 문제 정의

현재 사용자는 장비 등록과 그룹 설정을 완전히 별개의 흐름으로 처리해야 한다.

**현재 흐름의 문제:**
- 장비를 등록한 후, 그룹 관리 페이지로 별도 이동해 그룹에 장비를 할당해야 함
- 두 작업 간 컨텍스트 단절로 인해 신규 사용자가 혼란을 겪음
- "장비 등록 후 그룹 설정이 필요하다"는 사실 자체를 인지하지 못할 수 있음
- 장비가 그룹 미배정 상태로 방치될 가능성이 높음

## 목표

장비 등록 완료 직후, 인라인 위저드 형태로 그룹 설정을 연속으로 진행할 수 있게 해 UX 흐름을 개선한다.

## 사용자 시나리오

### 주요 흐름 (Happy Path)

```
1. [장비 관리] → "+ 장비 추가" 클릭
2. 장비 등록 완료
3. 모달 내 다음 단계: "그룹에 추가하시겠습니까?"
   ├── [예, 그룹에 추가] → Step 2로
   └── [나중에] → 모달 닫기, 완료
4. Step 2: 그룹 선택 방식
   ├── [기존 그룹에 추가] → 그룹 목록 표시 → 선택 → 완료
   └── [새 그룹 생성] → 그룹명/설명 입력 → 생성 → 완료
5. 완료 토스트: "장비 등록 및 그룹 배정 완료"
```

### 보조 흐름

- "나중에" 선택 시: 장비 카드에 "그룹 미배정" 뱃지 표시 (선택적 개선사항)
- 기존 그룹이 없을 경우: "기존 그룹에 추가" 버튼 비활성화 or 새 그룹 생성만 표시

## 기능 요구사항

### FR-01: 위저드 Step 구성
- Step 1 (장비 등록): 기존 DeviceRegistration 컴포넌트 (변경 없음)
- Step 2 (그룹 질문): "그룹에 추가하시겠습니까?" + [예] / [나중에]
- Step 3a (기존 그룹 선택): 기존 그룹 목록 → 선택 → 장비 할당 API 호출
- Step 3b (새 그룹 생성): 그룹명/설명 입력 → 그룹 생성 + 장비 할당 API 호출

### FR-02: 위저드 진행 표시
- 모달 상단에 현재 단계 표시 (예: 스텝 인디케이터 또는 단계 텍스트)
- "이전" 버튼으로 Step 2 → Step 1으로 돌아갈 수 있어야 함

### FR-03: 그룹 목록 필터링
- 기존 그룹 선택 시, 전체 그룹 표시 (장비 타입 무관)
- 그룹 없을 경우 "새 그룹 생성으로 이동" 안내

### FR-04: 완료 피드백
- 위저드 완료 시 통합 성공 토스트 표시
- 장비 목록 및 그룹 목록 자동 갱신

### FR-05: 권한 처리
- isFarmUser(농장 사용자)는 그룹 생성/수정 불가이므로 위저드에서 그룹 설정 단계를 건너뜀

## 비기능 요구사항

- 기존 DeviceRegistration 컴포넌트를 최대한 재사용
- 새로운 그룹 관련 상태는 GroupStore 활용
- 위저드 모달은 기존 모달 스타일 시스템 적용

## 수정 대상 파일 (예상)

| 파일 | 변경 유형 | 내용 |
|------|-----------|------|
| `components/devices/DeviceRegistration.vue` | 수정 | 위저드 step 추가 |
| `components/groups/GroupCreationModal.vue` | 재사용 or 추출 | 그룹 생성 폼 |
| `views/Devices.vue` | 소폭 수정 | 등록 완료 핸들러 |
| `stores/group.store.ts` | 재사용 | 그룹 목록 조회, 장비 할당 |

## 범위 제외 (Out of Scope)

- 장비 카드의 "그룹 미배정" 뱃지 (별도 이슈로 분리)
- 그룹 관리 페이지의 역방향 위저드 (그룹 생성 후 장비 추가 흐름은 현행 유지)
- 장비 타입별 그룹 필터링 (이번 버전에서는 전체 그룹 표시)

## 성공 기준

- 장비 등록 완료 후 페이지 이동 없이 그룹 설정 가능
- "나중에" 선택 시 기존 흐름과 동일하게 동작
- 기존 그룹 추가 / 새 그룹 생성 두 경로 모두 동작
- isFarmUser 권한 시 그룹 설정 단계 미노출
