## 2. Critical Issues (즉시 수정 필요)

### 2.1 [C-01] 모바일 모달이 뒷 배경 콘텐츠를 가리지 못함
- **위치:** farm/mobile/modal-device-register, modal-group-add, modal-group-add-device, modal-rule-edit, modal-rule-wizard
- **증상:** 모바일에서 모달이 full-screen으로 열리지 않고, 뒷 배경 콘텐츠가 스크롤되면서 겹쳐 보임
- **심각도:** HIGH
- **개선안:**
  - 모바일에서는 모달을 full-screen bottom sheet 또는 전체 화면 페이지로 전환
  - `body { overflow: hidden }` 추가로 배경 스크롤 방지
  - backdrop overlay를 더 불투명하게 (opacity 0.7 이상)

### 2.2 [C-02] "0개 추가" 버튼이 활성화 상태
- **위치:** admin/desktop/modal-group-add-device, admin/mobile/modal-group-add-device
- **증상:** 추가할 장비가 없는데 "0개 추가" 버튼이 클릭 가능한 상태
- **심각도:** HIGH
- **개선안:**
  - 선택된 항목이 0개일 때 버튼을 disabled 처리
  - 버튼 텍스트를 "장비 선택 후 추가" 등으로 변경

### 2.3 [C-03] 날짜 포맷 불일치
- **위치:** modal-harvest-batch-add (admin/farm 공통)
- **증상:** 파종일이 `mm/dd/yyyy` (미국식) 포맷으로 표시, 정식일은 `mm/dd/yyyy` placeholder 그대로
- **심각도:** MEDIUM-HIGH
- **개선안:**
  - 한국어 UI이므로 `yyyy-mm-dd` 또는 `yyyy년 mm월 dd일` 포맷 통일
  - 다른 페이지(수확 관리 배치 목록)에서는 `2026-04-01` 형식 사용 중 → 통일 필요

## 3. Major Issues (다음 스프린트 수정)

### 3.1 [M-01] 사이드바 메뉴 구조 차이 (Admin vs Farm)
- **Admin 메뉴:** 대시보드, 장비 관리, 그룹 관리, 자동화 룰, 환경 모니터링, 리포트, 수확 관리, 센서 알림, 사용자 관리 (9개)
- **Farm 메뉴:** 대시보드, 장비 관리, 그룹 관리, 자동화 룰, 환경 모니터링, 리포트, 수확 관리, 센서 알림 (8개)
- **이슈:** "사용자 관리"만 차이인데, 권한별 메뉴 표시는 올바름. 다만 Farm에서 메뉴가 하나 줄어도 사이드바가 동일한 높이를 차지하여 하단 여백이 큼
- **개선안:** 사이드바 하단에 빈 공간을 활용하여 빠른 액션 또는 시스템 상태 위젯 배치

### 3.2 [M-02] 대시보드 정보 밀도 차이
- **Admin 대시보드:** 날씨, 가동 장비 0/0, 센서 현황 0/0, 전체 장비 0, 활성 그룹 1, 자동화 룰 1/1, 온라인 기기 0 → 데이터 없는 상태가 대부분
- **Farm 대시보드:** 날씨, 가동 장비 1/5(상세 리스트), 센서 현황 1/1(실시간 데이터), 전체 장비 6, 활성 그룹 1, 자동화 룰 0/3, 온라인 기기 2 → 풍부한 데이터
- **이슈:** Admin에서 데이터가 없을 때 빈 카드가 너무 허전함
- **개선안:**
  - 빈 상태에 일러스트레이션 + "첫 장비를 등록해보세요" 같은 행동 유도(CTA) 추가
  - 대시보드 위젯에 "설정하기" 링크 추가

### 3.3 [M-03] 환경 모니터링 빈 상태 처리 부족
- **위치:** admin/desktop/sensors, admin/mobile/sensors
- **증상:** "센서가 등록된 그룹이 없습니다" 텍스트만 표시
- **개선안:**
  - 센서 등록 가이드 스텝(1. 장비 등록 → 2. 그룹 설정 → 3. 센서 환경 설정) 시각적 안내
  - 빈 상태 일러스트레이션 추가
  - "장비 관리로 이동" 바로가기 버튼

### 3.4 [M-04] 리포트 페이지 모바일 레이아웃
- **위치:** admin/mobile/reports, farm/mobile/reports
- **증상:**
  - 기간 선택 버튼이 세로로 쌓여 너무 많은 공간 차지 (12시간, 1일, 7일, 1개월, 기간 선택)
  - "다운로드" 라벨이 줄바꿈되어 "다운로\n드"로 표시됨
  - farm 모바일에서 상세 데이터 테이블이 가로 스크롤 없이 컬럼이 잘림
- **개선안:**
  - 기간 선택을 가로 스크롤 칩(Chip) 형태로 변경
  - 다운로드 영역을 아이콘 버튼으로 변경
  - 테이블을 카드형 리스트로 변환하거나 가로 스크롤 허용

### 3.5 [M-05] 자동화 룰 위저드 - 선택 없이 "다음" 버튼 활성화
- **위치:** admin/desktop/modal-rule-wizard, farm/desktop/modal-rule-wizard
- **증상:** 그룹을 선택하지 않은 상태에서 "다음" 버튼이 비활성화(회색)이지만, 선택 후에는 녹색으로 변경됨 → 이 부분은 올바름
- **개선점:** 비활성 상태일 때 "그룹을 선택해주세요" 같은 안내 텍스트 추가

### 3.6 [M-06] 센서 알림 페이지 - 알림 뱃지 위치
- **위치:** farm/desktop/alerts
- **증상:** 알림 뱃지(7)가 페이지 우측 상단에 떠 있어 보이는데, 헤더 영역과 정렬이 안 맞음
- **개선안:** 알림 뱃지를 "센서 알림" 제목 옆이나 탭 영역에 통합

## 4. Minor Issues (점진적 개선)

### 4.1 [m-01] 글자 크기 / 화면 모드 토글 위치
- **위치:** 모든 데스크톱 페이지 사이드바 하단
- **증상:** "글자 크기(가/가/가)"와 "화면 모드(밝게/어둡게)" 컨트롤이 있으나 모바일에서는 보이지 않음
- **개선안:** 모바일 햄버거 메뉴 내부 또는 설정 페이지에서 접근 가능하도록

### 4.2 [m-02] 장비 카드 디자인 불일치
- **위치:** farm/desktop/devices
- **증상:** 장비마다 카드 디자인이 다름
  - "석문리 개폐기" → 텍스트 기반 토글
  - "관수 TEST" → 상태/장비 태그 + 토글
  - "육묘장 휀", "석문리 왼쪽2동 휀" → 아이콘 없는 간단 카드
  - "석문리기상관측센서" → 보라색 배경 + 수치 표시
- **개선안:**
  - 장비 타입별 일관된 카드 템플릿 사용
  - 센서 카드의 보라색은 차별화에 좋으나, 다른 카드들도 타입별 엑센트 컬러 적용

### 4.3 [m-03] 그룹 관리 페이지 - 아이콘 버튼 의미 불분명
- **위치:** admin/desktop/groups, farm/desktop/groups
- **증상:** 그룹 카드 우측 상단의 아이콘 버튼들(톱니바퀴, +, 휴지통, 필터)이 label 없이 아이콘만 표시
- **개선안:**
  - 아이콘에 tooltip 추가
  - 또는 점(⋮) 메뉴로 통합하여 드롭다운에서 텍스트와 함께 표시

### 4.4 [m-04] 수확 관리 - 작업 추천 카드 반복
- **위치:** admin/desktop/harvest-rec, farm/desktop/harvest-rec
- **증상:** "하엽 제거", "유인 작업", "병충해 방제" 카드가 동일한 구조로 반복되는데, 각 카드의 "추천 이유"가 모두 동일한 내용("첫 작업 필요" 또는 "최근 6시간 고습...")
- **개선안:**
  - 작업별로 차별화된 추천 이유 제공
  - 긴급도/우선순위에 따른 시각적 차별화 (현재 모두 "정상" + 녹색)

### 4.5 [m-05] 배치 추가 모달 - 스크롤 필요
- **위치:** admin/desktop/modal-harvest-batch-add, farm/desktop/modal-harvest-batch-add
- **증상:** 모달 내 폼 필드가 많아(프리셋, 작물명, 품종, 그룹, 파종일, 정식일, 생육기간, 메모) 스크롤 필요
- **개선안:**
  - 모달 대신 페이지 내 인라인 폼 또는 드로어(Drawer) 사용 고려
  - 또는 2단계 위저드로 분리 (기본 정보 → 상세 정보)

### 4.6 [m-06] 센서 환경 설정 모달 - 모바일 full-screen 처리 양호
- **위치:** admin/mobile/modal-env-config, farm/mobile/modal-env-config
- **긍정적:** 모바일에서 full-screen으로 잘 처리됨
- **개선점:** "내부 환경" / "외부 환경" 섹션 구분이 모바일에서 시각적으로 약함 → 섹션 헤더에 배경색 추가

### 4.7 [m-07] 자동화 룰 - "동작: open" 텍스트
- **위치:** admin/desktop/automation
- **증상:** 동작 값이 "open"으로 영문 그대로 표시
- **개선안:** "열기" 또는 "개폐기 열기" 등 한국어로 표시

## 5. 모바일 전용 이슈

### 5.1 [mob-01] 모바일 헤더 "스마트팜" 텍스트 링크
- **위치:** 모든 모바일 페이지
- **증상:** 상단 "스마트팜" 텍스트가 초록색 링크 스타일로 표시되어 클릭 유도하나, 일관성 부족
- **개선안:** 로고 이미지 사용 또는 볼드 텍스트로 변경

### 5.2 [mob-02] 모바일 그룹 관리 - 액션 버튼 행 가독성
- **위치:** admin/mobile/groups, farm/mobile/groups
- **증상:** "0개 장비 ⚙ + 🗑 ▼" 아이콘 행이 모바일에서 너무 작고 터치 타겟이 좁음 (최소 44x44px 필요)
- **개선안:** 아이콘 크기 증가 또는 점(⋮) 메뉴로 통합

### 5.3 [mob-03] 모바일 환경 모니터링 - 데이터가 있는 경우
- **위치:** farm/mobile/sensors
- **증상:** 환경 모니터링 데이터가 풍부하지만 매우 긴 스크롤 필요 (원본 3868px 높이)
- **개선안:**
  - 접이식(Collapsible) 섹션 사용
  - "방울토마토 환경 종합" 점수(46점)를 상단에 고정하고, 세부 항목은 탭으로 분리

### 5.4 [mob-04] 모바일 리포트 - 차트 가독성
- **위치:** farm/mobile/reports
- **증상:** 온도/습도 추이 차트가 모바일에서 너무 작아 데이터 포인트 식별 어려움
- **개선안:**
  - 차트를 전체 너비로 확장
  - 가로 스크롤 가능한 차트 또는 터치로 데이터 포인트 확인 기능

## 6. 긍정적 평가 (잘 된 부분)

1. **컬러 시스템:** Green 기반 Primary 컬러가 농업 테마와 잘 어울리며 일관성 있게 적용됨
2. **자동화 룰 위저드:** 5단계(그룹→센서→장비→조건→확인) 스텝 표시가 직관적
3. **수확 관리:** 배치 카드의 생육 단계 태그(영양생장), 진행률 바, 다음 예정 작업 표시가 유용
4. **환경 모니터링 (Farm):** VPD, 환기 필요도, 온도 변화, UV 위험도 등 종합 점수 시스템이 우수
5. **글자 크기 / 다크 모드:** 접근성을 고려한 설정 옵션 제공
6. **센서 알림:** 활성 센서, 알림, 대기 목록 탭 구조가 명확
7. **Farm 대시보드:** 실시간 센서 데이터와 장비 상태를 한 화면에서 확인 가능
